on-device spelling + grammar

Type prose into the textarea below. Every time you stop typing for a moment, Dhamaka hands the whole sentence to an on-device instruction-tuned language model running inside this browser tab and asks it to rewrite the text with grammar and spelling mistakes fixed. The model's rewrite is then diffed against your original text word-by-word, and each change becomes a clickable chip. No rules, no hardcoded dictionary, no server — a real LLM reading and rewriting your prose.

Try some real sentences:
Because the model sees the whole sentence it can catch things a per-word spellchecker never could — subject-verb agreement ("I has" → "I have"), wrong articles ("a apple" → "an apple"), and real-word errors ("went their" → "went there") all get flagged in the same pass as plain misspellings.

warming up the model…
First visit on this device downloads a ~95 MB instruction-tuned language model (Xenova/LaMini-Flan-T5-248M). It's cached in your browser's IndexedDB forever after — every future visit is instant and works offline. 15–40 seconds on typical broadband, once.
—
The model runs through @huggingface/transformers, loaded lazily from esm.sh. Dhamaka wraps it behind the same task / SmartField / Transform API every other demo uses — the runtime underneath is pluggable, the product layer doesn't move.

what's happening under the hood

  oninput (debounced 400ms) → SmartText → runTask("spellcheck")
       │
       ▼
  spellcheckTask.slow(text, context, engine)
       │
       ├─ build prompt: "Fix the grammar and spelling … Text: {INPUT}"
       ├─ engine.complete(prompt, { maxTokens: 256, temperature: 0 })
       │    ↑ LaMini-Flan-T5-248M via Transformers.js (seq2seq, WASM)
       │
       ├─ clean the model reply (strip "Corrected:", drop quotes)
       │
       ├─ tokenize ORIGINAL + CORRECTED into word arrays
       ├─ LCS-align them at the word level
       └─ collapse each (delete+insert) pair into a { from, to, index }
          suggestion chip, skipping pure case/punctuation no-ops

  Nothing leaves the tab. No server, no API key, no rate limit.
  First visit downloads ~95 MB once, cached in IndexedDB forever.
  Per-call latency: ~250–800 ms for a short sentence on a laptop.

This is different from a traditional spellchecker in the one way that matters: the model sees the whole sentence in a single forward pass. A per-word masked-LM can only ever ask "is this specific word surprising at this specific position" — which misses every grammar error and every real-word error. A seq2seq model is trained to rewrite, so it catches both kinds of mistake in the same call. No rules, no special cases.

The model Dhamaka ships here is Xenova/LaMini-Flan-T5-248M: 248 M parameters, instruction-tuned on top of Flan-T5, under 100 MB quantised. You can swap it for any text2text-generation model Transformers.js supports — bigger models correct better but download slower. Same SDK, same task, same surface.
