Capability families

Not a raw model runtime. The product layer above it.

Dhamaka sits above window.ai, Transformers.js, WASM, and mock engines. Developers ship task-oriented UX today: fields that infer, textareas that understand context, contact blobs that become form data, and formulas that rewrite without leaking company data.

Reflex

Reactive, keystroke-level intelligence for inputs and textareas. Rules first, fuzzy second, model only when the fast path is uncertain.

SmartField SmartForm SmartText smart paste

Transform

One-shot instruction-driven rewrites for app data. The flagship case is erp.ai-style formula editing, explanation, and debugging.

formula-transform formula-explain formula-debug

Smart paste

Structured extraction for pasted contact blobs. Email, phone, website, social handles, names, and company hints land in the right fields without a server call.

paste-extract onpaste form fill

Formula tasks

Deterministic spreadsheet edits use structural rewrites first, then the same task shape can escalate when the instruction needs model reasoning.

Hero case

Copilot for sensitive formulas, without shipping formulas away.

ERP formulas contain pricing models, margins, payroll math, commission tiers, inventory rules, and compliance checks. Dhamaka lets products like erp.ai transform and explain those formulas in the tab where the data already lives.

Transform.formula() source: rule

A user selects a cell with a tax formula and types a plain-English instruction. The rules layer recognizes the discount pattern and rewrites structurally in microseconds. Weirder instructions can escalate to an on-device model.

input: =SUM(A1:A10) * 1.08
ask: add a 10% discount for employees
output: =(SUM(A1:A10) * 1.08) * 0.9
why: multiplied by 0.9 to apply the discount
Architecture

Every task runs through the fastest layer that can answer.

Dhamaka optimizes for a simple test: would this still work if the user's laptop had no network connection and no account with an AI provider?

your app

Install

One package. No provider account.

Install the public SDK and attach Dhamaka where your app already has intent: form fields, textareas, spreadsheet cells, data import screens, and internal tools.

npm package GitHub repo
quickstart.js dhamaka@0.1.0 
import { SmartForm } from "dhamaka";

const form = document.querySelector("#checkout");

new SmartForm(form, {
  tasks: { city: "city-to-state" },
  infer: {
    "city -> state": "city-to-state:stateName",
    "city -> country": "city-to-state:countryName",
    "city -> timezone": "city-to-state:tz",
    "city -> currency": "city-to-state:currency",
  },
});
FAQ

Local-first AI, built for product surfaces.

Is Dhamaka a model runtime?

Not primarily. It uses runtimes underneath, but the public value is the capability layer: tasks, reflexes, transforms, engine selection, and browser-native UX.

What ships today?

Reflex and Transform. That includes SmartField, SmartForm, SmartText, smart paste, formula transform, formula explain, and formula debug.

Why local?

Latency, privacy, cost, rate limits, and offline support all improve when the model runs where the app data already is.

What does the name mean?

Dhamaka means a burst or impact. Here, it is the moment an ordinary web form gets an on-device intelligence layer.
